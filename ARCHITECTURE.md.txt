# ARCHITECTURE.md

## SystemForge — Backend Architecture

---

## 1. System Overview

SystemForge is an enterprise-grade backend platform that allows developers to design, customize, and generate scalable system architectures for their applications.

It acts as a **System Design Builder**, enabling users to:

* Select backend systems (Auth, Payment, Notification, etc.)
* Customize configurations
* Generate production-ready architecture blueprints

The backend is designed as a **Modular Monolith**, with a clear path to microservices.

---

## 2. High-Level Architecture

```
Client (React / Web App)
        │
        ▼
API Gateway (Future)
        │
        ▼
Spring Boot Backend (Modular Monolith)
        │
 ┌─────────────────────────────────────────┐
 │            CORE MODULES                 │
 │                                         │
 │  Auth Module                            │
 │  User Module                            │
 │  System Module (Core Engine) ⭐          │
 │  Template Module                        │
 │  Recommendation Module                  │
 │  Notification Module                    │
 │  Admin Module                           │
 │                                         │
 │  Common (Shared utilities/config)       │
 └─────────────────────────────────────────┘
        │
        ▼
Database (MySQL → PostgreSQL later)
        │
        ▼
Future:
- Redis (Caching)
- Kafka (Event Streaming)
```

---

## 3. Module Architecture

Each module is isolated and follows clean architecture:

```
module/
├── controller
├── service
├── repository
├── entity
├── dto
├── mapper
```

### Modules

#### 1. Auth Module

* Handles authentication strategies
* Supports:

  * OTP
  * Email/Password
  * Social login (future)

#### 2. User Module

* User profile management
* Roles:

  * ADMIN
  * DEVELOPER

#### 3. System Module ⭐ (CORE)

* Main engine of the platform
* Responsible for:

  * System selection (Auth, Payment, etc.)
  * System configuration
  * User-defined architecture

#### 4. Template Module

* Stores predefined system templates
* Example:

  * OTP Auth for ride-hailing apps
  * JWT Auth for SaaS apps

#### 5. Recommendation Module

* Suggests best systems based on:

  * App type
  * Scale
  * Features

#### 6. Notification Module

* Sends:

  * Emails
  * Push notifications
  * System alerts

#### 7. Admin Module

* Manage templates
* Manage users
* Monitor usage

#### 8. Common Module

* ApiResponse
* Exceptions
* Configurations
* Utilities

## 4. Core Engine (System Builder)

### Flow

```
User selects:
  App Type (e.g., Ride App)

        ↓

User selects systems:
  - Auth
  - Payment
  - Notification

        ↓

User customizes:
  - Auth Type = OTP
  - Payment = Razorpay

        ↓

SystemForge Engine processes:
  - Applies rules
  - Matches templates
  - Generates architecture

        ↓

Output:
  - API design
  - DB schema
  - Tech stack
  - System explanation
```
